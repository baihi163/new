# 任务二报告：神经网络拟合实验与学习率分析

## 实验概述
本实验旨在通过构建一个简单的全连接神经网络（MLP），对给定的数据进行函数拟合，并分析学习率（Learning Rate, LR）对训练过程和模型性能的影响。实验使用 PyTorch 框架，数据集来源于 `data.csv`，模型通过均方误差（MSE Loss）作为损失函数，采用 Adam 优化器进行训练。训练轮数设置为 1000 个 Epoch，隐藏层神经元数量为 20。

实验主要测试了三种学习率：
- **LR = 0.05**（基准值）
- **LR = 1.0**（非常大）
- **LR = 0.000001 (1e-06)**（非常小）

以下是实验结果的详细分析，包括学习率对 Loss 曲线和拟合效果的影响。

---

## 实验结果与可视化

### 1. 学习率 = 0.05（基准值）
- **Loss 曲线**：
  - Loss 从初始值约 1.8 x 10² 逐步下降到约 1.4 x 10²。
  - 具体数据（终端输出）：Loss 从 Epoch 10 的 164.644180 下降到 Epoch 1000 的 143.882126（或另一组输出的 134.756882）。
  - Loss 曲线在对数坐标下呈现平滑下降趋势，表明模型逐步收敛。
- **拟合效果**：
  - 在 Epoch=10 时，预测曲线与真实数据点（Ground Truth）有较大偏差。
  - 随着训练进行，到 Epoch=100 和 Epoch=1000 时，预测曲线逐步贴近真实数据点，拟合效果显著改善。
- **结论**：
  - 学习率 0.05 是一个较为合理的选择，模型能够在有限轮数内有效学习数据特征，Loss 下降明显，拟合效果较好。
- **可视化结果**：
  ![Function Fitting and Loss Curve for LR=0.05](Figure_1.png)

### 2. 学习率 = 1.0（非常大）
- **Loss 曲线**：
  - Loss 值在极高范围内波动（从 10³ 到 10⁵），无明显下降趋势。
  - 具体数据（终端输出）：Loss 从 Epoch 10 的 199.256317 下降到 Epoch 1000 的 161.396347，但整体仍维持较高水平，且波动较大。
  - Loss 曲线在对数坐标下呈现剧烈震荡甚至上升趋势，表明训练过程不稳定。
- **拟合效果**：
  - 预测曲线（Epoch=10, 100, 1000）完全偏离真实数据点，呈现无意义的形状。
  - 即使到 Epoch=1000，预测仍未贴近数据点，拟合效果极差。
- **结论**：
  - 学习率 1.0 过大，导致模型训练发散，无法有效学习数据模式。
- **可视化结果**：
  ![Function Fitting and Loss Curve for LR=1.0](Figure_2.png)

### 3. 学习率 = 0.000001 (1e-06)（非常小）
- **Loss 曲线**：
  - Loss 值几乎不变，从初始值 1.836 x 10² 仅微降到 1.832 x 10²。
  - 具体数据（终端输出）：Loss 从 Epoch 10 的 183.597473 仅下降到 Epoch 1000 的 183.194427，变化极小。
  - Loss 曲线在对数坐标下几乎是一条平线，表明模型几乎没有优化。
- **拟合效果**：
  - 预测曲线（Epoch=10, 100, 1000）几乎重叠，且与真实数据点有较大偏差。
  - 即使到 Epoch=1000，预测曲线仍接近初始随机预测，拟合效果极差。
- **结论**：
  - 学习率 0.000001 过小，模型在有限轮数内无法显著调整参数，拟合效果极差。
- **可视化结果**：

## 学习率分析

**问题**：当你把学习率（Learning Rate）调得非常大（例如 1.0）或非常小（例如 0.00001）时，Loss 曲线发生了什么变化？为什么？

**回答**：
- **学习率非常大（LR = 1.0）**：
  - **Loss 曲线变化**：Loss 曲线呈现剧烈震荡，值在较高范围（10³ 到 10⁵）波动，甚至不下降反而上升（终端输出：从 199.256317 到 161.396347，仍较高）。在对数坐标下，曲线无稳定下降趋势，表明训练发散。
  - **原因**：学习率过大时，参数更新步长过长，优化器在参数空间中“跳跃”过大，容易越过损失函数的最优解（全局或局部极小值），导致模型无法收敛。极端情况下，参数值可能超出合理范围，造成数值不稳定甚至 Loss 变为 `NaN`。在本次实验中，Adam 优化器的自适应机制可能缓解了一部分发散，但仍无法克服过大学习率带来的不稳定性。
- **学习率非常小（LR = 0.000001 或 1e-06）**：
  - **Loss 曲线变化**：Loss 曲线几乎不变，仅从 1.836 x 10² 微降到 1.832 x 10²（终端输出：从 183.597473 到 183.194427）。在对数坐标下，曲线接近一条平线，表明模型几乎未优化。
  - **原因**：学习率过小时，参数更新步长极短，每次迭代只能进行微小调整。优化器需要极多的迭代次数才能接近损失函数的最优解。在有限训练轮数（1000 Epochs）内，模型参数几乎没有显著变化，无法充分拟合数据。对于复杂数据分布（如非线性关系），小学习率还可能导致模型陷入梯度平坦区域，难以进一步优化。
- **数学原理**：
  - 损失函数优化过程是在参数空间中寻找最低点（极小值）。学习率决定了每次沿梯度反方向移动的步长。过大的学习率导致步长过长，容易越过最低点；过小的学习率导致步长过短，移动缓慢。
  - 本实验的损失函数为均方误差（MSE Loss），其公式为：
    $$ \text{Loss} = \frac{1}{n} \sum_{i=1}^n (y_i - \hat{y}_i)^2 $$
    其中 $$y_i$$ 是真实值，$$\hat{y}_i$$ 是模型预测值。学习率影响模型预测值 $$\hat{y}_i$$ 的更新速度和准确性。

## 结论与建议
- **结论**：
  - 学习率对神经网络训练的收敛性和拟合效果有显著影响。合理的学习率（如 0.05）能够使 Loss 平滑下降，模型逐步拟合数据。
  - 过大的学习率（如 1.0）导致 Loss 震荡和训练发散，拟合效果极差；过小的学习率（如 1e-06）导致 Loss 下降缓慢，模型几乎未优化。
- **建议**：
  - 当前实验中，学习率 0.05 是较优选择。未来可进一步尝试 0.01 到 0.1 之间的值（如 0.02 或 0.08），观察是否能更快收敛或获得更好拟合效果。
  - 可引入学习率调度机制（如 `torch.optim.lr_scheduler.StepLR`），在训练后期逐步降低学习率，帮助模型更精细地接近最优解。
  - 增加训练轮数（EPOCHS）或调整隐藏层神经元数量（HIDDEN_SIZE），观察是否能进一步提升拟合效果。
